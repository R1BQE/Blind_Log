Program Description

Blind_Log is designed for keeping a log of radio contacts (QSO). It allows you to add, edit, delete, and export QSO records in ADIF format.

Program Interface

When the program opens, the focus is on the "Add QSO" tab. In this tab, you can enter data about a radio contact, such as callsign, name, city, QTH, frequency, RST received, RST sent, mode, band, and comment.

Use Ctrl+Tab and Ctrl+Shift+Tab to switch between the "Add QSO" and "Journal" tabs.

"Add QSO" tab:

  • Focus is by default on the "Callsign" field.
  • Use the Tab key to move between input fields: "Callsign", "Name", "City", "QTH", "Frequency", "RST Received", "RST Sent", "Mode", "Band", "Comment", "Date", "Time", and the "Add QSO" button.
  • Shift+Tab allows moving in reverse order.

"Journal" tab:

  • Focus is by default on the list of QSO records.
  • Use Tab to move between the list and the "Edit" and "Delete" buttons.
  • Shift+Tab allows moving in reverse order.

To export the journal, use the hotkey Ctrl+S on the "Journal" tab.

Interface Settings

Open settings (Ctrl+P). The settings dialog has two tabs, switchable with Ctrl+Tab and Ctrl+Shift+Tab.

"General" tab:

  • Operator callsign, name, QTH, city, equipment (for QSO window info).
  • Coordinates (latitude, longitude) for ADIF export.
  • Timezone settings (UTC or custom hour range).
  • Option to search callsigns on QRZ.ru with login and password.
  • Logging options and check for updates on startup.

"Interface" tab:

  • Visibility checkboxes for fields: callsign (required), name, city, QTH, frequency, band, mode, RST received, RST sent, comment, date, time.
  • By default, all fields are visible. Uncheck to hide a field in the add QSO form and journal columns.
  • Note: The "Callsign" field is always visible and required. So there's no checkbox for it in interface settings.
  • When hiding "Date" and "Time" fields, the system automatically uses current date and time (considering timezone from settings) when adding QSO and exporting to ADIF.
  • Interface language: Select the program language from the list: "System default" (automatically detects your OS language), "English", or "Русский". After changing the language, click "Save" and restart the program for changes to take effect.

After changing field visibility, click "Save" — the interface will update immediately.

Interface Language

Blind_Log supports multiple interface languages to make it convenient for users from different countries.

How to change language:

  • Press Ctrl+P to open settings.
  • Go to the "Interface" tab (second tab).
  • In the "Interface language" section, select the desired language from the list:
    • System default — the program will automatically choose the language based on your operating system settings (e.g., Russian if the system is in Russian).
    • English — English language.
    • Русский — Russian language.
  • Click "Save" to apply changes.
  • Important: After changing the language, restart the program so all texts update.

If you're using the program for the first time, try selecting the language that's most comfortable for you. "System default" is set by default.

Required Fields

When adding a QSO, the "Callsign" field must be filled. Other fields are optional. If any optional fields are hidden in the interface, they won't be shown in the form but may be filled automatically by the program (e.g., date and time if they are hidden).

Russian Text Transliteration

The program supports automatic transliteration of Russian text to Latin. This is needed for compatibility with international logging systems (LoTW, Club Log, etc.), which use the Latin alphabet.

How it works:

  • When adding a QSO, Russian text in the Name, City, and Comment fields is automatically converted to Latin.
  • Example: "дмитрий" → "dmitrij", "москва" → "moskva".
  • The Callsign field remains unchanged (usually already contains letters and numbers).
  • Transliteration only happens if the text contains Cyrillic. English text remains unchanged.
  • When exporting to ADIF, all these fields contain the transliterated text.

This feature is enabled by default and requires no additional settings. It triggers automatically when entering Russian text.

Saving Journal on Exit

If there is at least one record in the journal, when trying to exit, the program will offer to save the journal in ADIF format, exit without saving, or cancel exit.

  • Save — opens the ADIF save dialog, after successful export the program closes.
  • Don't save — the program closes without saving the journal.
  • Cancel — exit is canceled, you can continue working.

Auto-save Session (temp)

The program supports automatic saving of the current work session. This helps recover data if the program crashes or the computer restarts.

How to enable:

  • Open settings (Ctrl+P).
  • On the "General" tab, check "Auto-save session (temp)".
  • Click "Save".

How it works:

  • When auto-save is enabled, the program automatically copies all QSO records to a hidden file blind_log_temp.json next to the executable.
  • This happens every time you add or delete a QSO.
  • The file updates in real-time, so even on a crash, only data that didn't get added is lost.

Recovery after crash:

  • On the next program start, if auto-temp is enabled and the temp file has data, you'll be prompted to restore the session.
  • The dialog shows the number of saved QSOs: "Found unsaved data (N QSOs). Restore?"
  • Click "Yes" to restore or "No" to start fresh.

Clearing temp:

  • After successful ADIF export, the temp file is automatically deleted (to avoid offering to restore empty data).
  • If you restored data, the temp file is also cleared to avoid asking about restoration twice.

Important: Auto-save protects against crashes, not a replacement for regular export. It's recommended to export the journal to ADIF after each logging session for a reliable backup.

Hotkeys

  • Ctrl+P - Open settings
  • Ctrl+Q - Exit program
  • Ctrl+Enter - Add QSO
  • Ctrl+E - Edit selected QSO
  • Ctrl+S - Export QSOs to ADIF (only via hotkey)
  • Delete - Delete selected QSO
  • Shift+F1 - About
  • F1 - Help
  • Ctrl+F1 - Shows changelog - program change log
  • Ctrl+Tab - Switch tabs forward (main window and settings)
  • Ctrl+Shift+Tab - Switch tabs backward (main window and settings)
  • Ctrl+U - Check for updates
  • Enter - Fill name and city by callsign from QRZ.ru (if setting enabled)

    • Keyboard shortcuts for quick navigation

    For quick filling of the QSO form, Alt key shortcuts are provided. They allow fast navigation to fields or cycling through combobox values.

    • Alt+C — focus Callsign field
    • Alt+N — focus Name field
    • Alt+T — focus City field
    • Alt+Q — focus QTH field
    • Alt+F — focus Frequency field
    • Alt+R — focus RST received field
    • Alt+S — focus RST sent field
    • Alt+M — cycle Mode
    • Alt+B — cycle Band
    • Alt+O — focus Comment field
    • Alt+D — focus Date field
    • Alt+I — focus Time field

    Each press selects the next value. After the last item, it returns to the first.
